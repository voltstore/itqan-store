# اتقان (Itqan) — متجر قطع الكمبيوتر والتجميعات الاحترافية

نموذج أولي عالي الدقة (High-Fidelity Prototype) — HTML/CSS/JS خام بدون أي frameworks أو build tools.

**v3**: هوية **أبيض + برتقالي + بني** — فاتح دافئ افتراضي + ثيم داكن «إسبريسو»،
**واجهة ثنائية اللغة (عربي RTL / إنجليزي LTR)**، أنيميشن سكرول، ولوحة تحكم كاملة
(إحصائيات + بحث شامل + إعدادات المتجر).

## التشغيل

افتح `index.html` مباشرة في المتصفح. لا يحتاج أي تثبيت.

- **الثيم**: الفاتح (أبيض دافئ) افتراضي، والداكن «إسبريسو» من زر الشمس/القمر.
- **اللغة**: العربية افتراضية، والتبديل من زر `English/عربي` في الهيدر (يُحفظ في localStorage).
  كل النصوص في `i18n.js` — أضف مفتاحًا هناك ثم استخدم `data-i18n` أو `L()`.
- **رقم الواتساب**: يُدار من لوحة التحكم `admin.html` (قسم إعدادات المتجر) — يُحفظ في `settings.js`.

## الإعداد المطلوب قبل الإطلاق

1. **رقم الواتساب**: من لوحة التحكم `admin.html` → «إعدادات المتجر» (يُحفظ في `settings.js`).
2. **مساعد الذكاء الاصطناعي**: من نفس نافذة الإعدادات — اختر المزوّد
   (Google Gemini / OpenAI ChatGPT / Anthropic Claude) + الموديل + المفتاح.
   يُحفظ في `settings.js` ويعمل **لكل الزوار** بلا إدخال منهم.
   - Gemini (مجاني): https://aistudio.google.com/app/apikey
   - OpenAI: https://platform.openai.com/api-keys
   - Anthropic: https://console.anthropic.com/settings/keys
   - ⚠️ **أمنياً**: المفتاح في `settings.js` يصل لكل زائر ويظهر في مصدر الصفحة —
     استخدم مفتاحًا بحد إنفاق محدود. الحل الآمن الكامل (وسيط خادم يخفي المفتاح) يأتي في مرحلة Firebase.

## لوحة التحكم (admin.html) — داشبورد كامل

واجهة تحكّم احترافية بشريط جانبي + جدول بيانات تفصيلي:

1. افتح `admin.html` في Chrome أو Edge.
2. اضغط **«اربط مجلد المشروع»** (أسفل الشريط الجانبي) واختر مجلد `ittcan store` — مرة واحدة لكل جلسة.
3. من الشريط الجانبي اختر الفئة (أو **«كل القطع»**) → **«＋ إضافة قطعة»**.
4. لكل قطعة: تعديل ✎ / نسخ ⧉ / حذف 🗑 — والجدول يعرض كل مواصفة في عمود مستقل.
5. أدوات: بحث شامل بالاسم/المعرّف، فرز بالسعر أو الاسم، وإحصائيات حيّة (العدد، القيمة، تغطية الصور والأوصاف).
6. **إعدادات المتجر** (زر ⚙︎ بالشريط الجانبي): رقم الواتساب → يُحفظ في `settings.js`.
7. الحفظ تلقائي: `inventory.js` + الصور تُنسخ إلى `images/<id>.<ext>`. بدون ربط المجلد استخدم «تنزيل نسخة».

## قواعد التوافق المطبّقة

1. سوكِت المعالج ↔ سوكِت اللوحة الأم (AM4 / AM5 / LGA1700)
2. نوع الذاكرة DDR4/DDR5 ↔ دعم اللوحة الأم
3. قدرة مزود الطاقة ≥ (استهلاك المعالج + كرت الشاشة + 100 واط) × 1.2 هامش أمان
4. الصندوق يدعم مقاس اللوحة (ATX / mATX)
5. المبرد يغطي حرارة المعالج (TDP)

## Firebase (السحابة) — مفعّل

المشروع مربوط بـ Firebase: **Realtime Database** (المخزون + الإعدادات)،
**Storage** (صور المنتجات)، و **Auth** (تسجيل دخول المشرف لحماية اللوحة).

- الإعداد الكامل خطوة بخطوة في **[FIREBASE-SETUP.md](FIREBASE-SETUP.md)**.
- الصق إعداد مشروعك في `firebase-config.js`. إن تركته بالقيم الافتراضية، يعمل
  الموقع محليًا من `inventory.js` / `settings.js` تلقائيًا (بدون Firebase).
- `data.js` → `getParts()` و `getSettings()` يقرآن من Firebase إن كان مضبوطًا،
  وإلا يرجعان للبيانات المحلية — نفس الواجهة، مسارَان.
- لوحة الأدمن في وضع Firebase تتطلب تسجيل دخول، وتكتب مباشرة للسحابة، وترفع الصور
  إلى Storage، وفيها زر «رفع البيانات المحلية إلى Firebase» للبذرة الأولى.
- قواعد الأمان: قراءة عامة + كتابة للمصرّح فقط (موجودة في FIREBASE-SETUP.md).
